# Kanad Theme Store Launch Plan

## 1. Publish repository and docs

1. Create a GitHub repository for this project.
2. Push the codebase to the default branch.
3. Enable GitHub Pages from the `docs/` folder.
4. Confirm that `index.html`, `merchant-guide.html`, `faq.html`, and `support.html` are publicly reachable.
5. Confirm that `config/settings_schema.json` matches the live documentation domain.

## 2. Finish support workflow

1. Connect `docs/support.html` to a production form backend.
2. Enable an automatic confirmation email for support requests.
3. Route support submissions into your ticketing or email workflow.
4. Document your response-time policy and escalation path.

## 3. Build demo stores

1. Create at least one development store for each preset you intend to submit.
2. Add realistic products, imagery, collections, articles, discounts, subscriptions, and gift cards.
3. Verify that install-state content matches the demo-store presentation.
4. Remove any placeholder copy or unsupported assets.

## 4. Validate storefront quality

1. Run Theme Check on the current branch.
2. Run Lighthouse on home, product, and collection pages for desktop and mobile.
3. Run accessibility checks and manual keyboard testing.
4. Test supported browsers and mobile webviews.
5. Validate checkout, cart, subscriptions, gift cards, and predictive search with real store data.

## 5. Complete Theme Store submission assets

1. Finalize theme and preset naming.
2. Confirm originality against Theme Store policy.
3. Finalize merchant-facing documentation and FAQ.
4. Finalize support contact form and response workflow.
5. Prepare release notes for the submission version.

## 6. Submit through Shopify Partners

1. Upload the validated theme zip or connect the repository from your Partner account.
2. Add documentation and support URLs to the listing.
3. Provide demo store access and theme metadata.
4. Submit for review.
5. Track reviewer feedback and turn fixes into versioned updates.
